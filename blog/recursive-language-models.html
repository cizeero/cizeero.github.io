Recursive Language Models

Blog Home

This post is about the paper Recursive Language Models by Zhang, Kraska, and Khattab. The core idea is elegant: instead of forcing an LLM to ingest an entire long context at once, let it programmatically examine, decompose, and recursively call itself over snippets of the input. The model writes code to selectively access and process portions of text dynamically, enabling it to handle inputs up to 100x longer than its native context window.

I used this concept in an agentic system with a summarization hybrid approach, making previous chats searchable. The agent can recursively examine conversation history, summarizing and indexing as needed, then query specific portions when relevant context is required. When I read about it, the idea seemed so obvious, yet no one made it for such a long time.
